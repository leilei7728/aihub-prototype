## 一、产品概述

### 1.1 产品定位

My tokens 是一个给 AI 高频用户使用的账号和计费管理页面，主要解决两件事：一是看清自己还能不能继续调用，二是把平台能力接到常用工具里。

这个系统主要提供以下能力和服务：

1. 开箱即用的工具服务  
用户下载安装已经预配置好的 AI 工具后，可以直接打开使用，不需要自己处理复杂安装和环境配置。

2. 平台内置模型直用能力  
进入系统后，用户可以直接使用已经配好的内置模型，无需自己找 Key、配接口或调参数。

3. 外部模型接入与协议转换能力  
如果用户不想在平台购买，也可以接入自己的模型供应商，继续使用原来的工具；用户只需要提供自己的 API Key，系统负责接入配置、地址适配和协议转换。

4. 账号与计费管理服务  
用户可以清楚看到订阅套餐可使用额度、充值余额，以及通过本平台路由的 tokens 消耗统计。

### 1.3 目标用户

| 用户类型 | 描述 | 主要需求 |
|---|---|---|
| AI 高频用户 | 长时间使用 Claude、Codex、OpenCode 等工具 | 稳定调用、统一计费、快速补能 |
| 轻中度体验用户 | 初次接触平台，想低门槛试用 | 低价试用、简单充值、快速开通 |
| 安装困难用户 | 不熟悉工具安装和环境配置，希望拿到就能用 | 下载已预配置安装包，开箱即用 |
| 多工具切换用户 | 同时使用多个桌面端、CLI、插件类工具 | 统一 Key、统一入口、统一消耗查看 |
| 协议不兼容用户 | 想在 Claude 等国外工具里使用国产模型，但模型原始接口协议不支持 | 通过平台完成协议转换后继续使用熟悉的工具 |

## 四、核心业务模型

### 4.0 两条业务主线

系统一期主要有两种使用方式。

#### 主线一：平台内置模型直用

- 用户下载 My tokens
- 用户通过 My tokens 直接下载已配置好的 AI 工具
- 用户在发起订阅或充值前登录账号
- 用户在平台内选择订阅套餐，或直接充值人民币余额
- 平台内置模型已完成协议适配、路由和调用配置
- 用户无需单独配置模型供应商信息
- 订阅或充值成功后，用户即可在工具中直接选择平台内置模型并发起调用

这条路径适合直接买平台能力然后马上开用的用户。

#### 主线二：外部模型接入使用

- 用户下载 My tokens
- 用户通过 My tokens 直接下载已配置好的 AI 工具
- 用户不买平台套餐，也不充平台余额
- 用户在新增供应商前登录账号
- 用户通过填写外部模型供应商信息并完成验证
- 系统拉取该 API Key 支持的模型并完成协议适配
- 用户可以在 AI 工具中选择外部供应商模型来使用

这条路径适合不买平台额度、只想接自己外部模型的用户。

### 4.1 双计费模式

系统采用两套并存的计费逻辑，两者都可直接用于平台内置模型。

#### A. 订阅制

- 用户购买固定周期套餐
- 套餐内明确包含可使用模型范围和对应使用次数
- 套餐按“每日发放调用次数”计量
- 每发起一次请求就扣减一次或多次订阅今日可使用次数
- 不同模型按 1x / 2x / 4x 扣减
- 当天未使用完的次数次日 0:00 清零
- 订阅今日可使用次数用完后，若账户仍有可使用余额，则自动切换到按量扣费

#### B. 按量充值

- 用户充值人民币余额
- 余额永久有效，不过期
- 每次调用按模型实际 API 单价扣减人民币余额
- 单价区分输入价格与输出价格


#### C. 外部模型接入

- 用户不买平台套餐、也不充平台余额，也可以接自己在外面买的模型
- 用户填写供应商名称、官网地址、请求地址、API Key 后发起验证
- 验证通过后，系统拉取该 API Key 当前支持的模型列表
- 系统负责做协议适配，使已拉取模型可在不同 AI 工具中被统一调用
- 该路径下的扣费由外部供应商自行完成，不计入平台订阅今日可使用次数和平台可使用余额

### 4.2 扣费优先级

系统默认扣费顺序：

1. 优先扣减订阅今日可使用次数
2. 订阅今日可使用次数不足时自动切换到可使用余额
3. 若订阅与余额均不足，则拦截请求并提示充值

### 4.3 平台接入口径

平台能力有三种使用方式：

- 在 My tokens 平台内启用和调用平台内置模型时，无需用户手动输入平台 API Key
- 在第三方 AI 工具中使用平台内置模型时，用户使用平台 API Key 与 Base URL 完成接入
- 在第三方 AI 工具中接入外部供应商模型时，用户提供自己的供应商 API Key，平台负责完成地址适配、接入配置和协议转换

- OpenAI 协议地址： https://api.aihub.ai/v1
- Anthropic 协议地址： https://api.aihub.ai/anthropic

平台自动根据当前用户订阅状态、余额状态和调用模型完成扣费判断，用户无需自己切换扣费模式；外部供应商模型不走平台计费，但仍通过 My tokens 完成验证、协议适配和工具侧配置。

## 六、页面需求

### 6.1 P01 Tokens 管理

#### 页面目标

让用户进入系统后第一时间知道还能不能继续用、最近消耗了什么，以及额度不够时去哪里续费或充值。

#### 页面内容

1. 顶部余额信息
- 展示订阅今日可使用次数
- 展示可使用余额
- 两块信息均可点击跳转购买页对应 Tab

2. 游客态 Hero
- 展示产品价值说明
- 展示 AI 工具卡片区，支持下载并直接打开使用
- 引导用户登录 / 注册

3. 已登录数据区
- AI 工具卡片区（展示工具使用概览）
- 数据图表区包含 3 个 Tab：模型总览、供应商总览、消耗明细
- 时间维度筛选：今天、近 7 天、近 30 天、全部
- 图表展示总消耗与区间汇总，并分别按模型、供应商、明细三个维度查看
- 单天视图按小时展示，近 7 天 / 近 30 天 / 全部按天展示
- 图表下方提供模型或供应商汇总卡片，点击卡片可筛选显示对应数据

#### 关键规则

- 游客也可下载并使用 AI 工具，但不展示真实个人数据
- 已登录后展示真实统计数据
- 消耗统计需与订单、扣费、成就体系中的用量口径保持一致
- 模型总览和供应商总览都支持“点击下方卡片筛选图表”的交互
- 卡片颜色需和上方图表里的对应系列颜色保持一致，帮助用户快速建立映射
- 当只看单天数据时，模型总览和供应商总览统一按小时展示，避免只出现单根柱状图

### 6.2 P02 AI 工具管理

#### 页面目标

让用户清楚知道平台支持哪些工具、每个工具当前用什么模型，以及需要更细配置时去哪里调整。

#### 页面内容

- 工具列表 / 工具卡片
- 工具启用状态
- 当前工具已选择模型
- 当前工具的供应商模型精细化配置入口
- 供应商模型在各工具中的启用情况

#### 功能要求

- 展示可下载的已预配置 AI 工具
- 展示工具下载状态，并支持下载后直接打开 AI 工具
- 展示工具当前已启用的模型
- 展示工具是否已完成接入配置
- 支持用户在工具维度对模型做启用、停用和编辑
- 对接 Claude 等工具时，需考虑模型槽位限制与差异化配置
- 当模型供应商页面最多选择 4 个模型仍不够用时，用户可在 AI 工具管理页面继续新增模型和做更多高级配置
- 提供前往安装、配置或管理的操作入口

#### 关键规则

- P02 负责单独针对这个工具的供应商模型精细化配置，不负责新增供应商
- 在 P03 新增成功的供应商，默认可在所有 AI 工具中显示并可被选择使用
- 若不同工具需要不同模型组合，用户可在 P02 按工具单独编辑
- P02 承接 Claude 之外其他工具的扩展模型配置和高级配置功能

### 6.3 P03 模型供应商

#### 页面目标

让用户直观看到系统内置供应商和外部供应商的使用情况，并以最少配置成本完成供应商接入。

#### 页面内容

- 系统内置供应商概览
- 已连接供应商列表
- 每个供应商的余额、今日消耗、本月消耗
- 添加供应商按钮
- 供应商连接状态展示

#### 单条供应商信息建议字段

- 供应商名称
- 官网地址
- 请求地址
- 账户余额 / 今日消耗 / 本月消耗
- 连接状态
- 最近检测时间
- 可用模型数量
- 编辑 / 删除 / 重新检测操作

#### 添加供应商配置项

说明：供应商配置属于账号资产，需登录后才能新增、编辑和保存；相关配置随账号同步。

- 供应商名称，必填
- 官网地址，选填
- 请求地址，必填
- API Key，必填

#### 模型拉取与选择规则

- 供应商信息验证通过后，系统自动拉取该 API Key 当前支持的全部模型
- 拉取到的模型会在所有 AI 工具中显示并可使用
- 在“模型供应商”新增流程中，最多允许用户直接选择 4 个模型
- 该 4 个模型上限主要用于适配 Claude 的模型槽位限制
- 若用户需要使用更多模型，可进入“AI 工具管理”页面，在编辑中继续新增模型并完成更多高级配置
- 系统自动完成协议适配和各工具基础配置，用户只需要选择要启用的模型即可

#### 关键规则

- 供应商接入能力主要用于扩展模型来源，不影响平台自己的 API Key 体系
- 新增成功的供应商默认在所有 AI 工具中显示并可使用
- 外部供应商模型调用消耗不计入平台订阅今日可使用次数和平台可使用余额

### 6.6 P07 我的成就

#### 页面目标

让用户看到自己的成长记录，并愿意继续使用和分享。

#### 页面内容

- 侧边栏头像勋章挂件
- 账号头像与成就标题
- 累计加入时长、已点亮成就数、当前进度完成度
- 勋章列表

#### 成就类型

### 6.7 P08 个人中心

P08 负责收拢与用户本人强相关的管理项。

#### A. 账号资料

- 展示头像、昵称、手机号
- 头像区域展示当前佩戴勋章，并与侧边栏头像挂件保持一致；默认佩戴近期获得的最新一枚勋章
- 展示当前订阅状态和有效期
- 展示今日已用次数、剩余次数、充值入口
- 展示续费提示与购买跳转

#### B. 我的成就

- 与 P07 共享成就内容，但作为个人中心内嵌视图承接

#### C. API Key

##### 页面目标

把平台 API Key 的用途和外部供应商接入边界讲清楚，并明确告诉用户：平台内调用不用手动填 Key。

##### 页面内容

- 平台 API Key 展示
- 显示 / 复制 / 重置 Key
- OpenAI / Anthropic 协议 Base URL
- 常见工具快速接入指引
- 计费说明
- 平台内调用无需填写 Key 的说明

##### 关键规则

- API Key 已生成，但若无有效订阅且余额不足以支付本次调用，则平台内置模型不可用
- 用户订阅或充值后 Key 立即恢复可用
- 在 My tokens 平台内调用内置模型时，无需用户手动输入 API Key
- 平台 API Key 主要用于在第三方 AI 工具中接入平台内置模型；外部供应商模型仍由用户提供自己的供应商 API Key
- 同一个 Key 可用于多种工具
- Key 重置后，系统生成新的 Key；用户需重新启用工具，原工具中已登录的旧 Key 立即失效
- 平台自动判断本次请求走订阅今日可使用次数还是可使用余额扣费

#### D. 通知设置

##### 通知项

- Tokens 不足提醒
- 路由异常提醒
- 成就升级提醒

##### 页面目标

把影响继续使用和留存的关键提醒独立出来，避免与资料编辑混杂。

## 七、关键业务规则

### 7.1 登录权限

- 未登录用户可浏览产品结构、下载 AI 工具并直接打开使用，但不展示真实个人数据，也不能保存任何与账号绑定的信息
- 以下能力需登录后使用：
  - 查看个人真实用量
  - 订阅 / 充值
  - 查看订单
  - 管理 API Key
  - 管理供应商
  - 查看个人成就与通知设置
- 原则：凡是需要收集、保存、回写或长期跟随用户账号的信息，均需在登录后操作

### 7.2 购买与可用性联动

- 有有效订阅且订阅今日可使用次数足够时，用户可按订阅额度发起调用
- 无有效订阅，或订阅今日可使用次数不足，但余额足以支付本次调用时，用户可按量发起调用
- 无有效订阅，且余额不足以支付本次调用时，平台内置模型不可调用
- 即使平台订阅与余额都不可用，只要已经接好外部供应商，用户仍可调用外部模型

### 7.3 退款规则

- 订阅订单支持 7 天内退款
- 已使用天数按日折算扣除
- 可使用余额仅支持未使用余额退款

### 7.4 开票规则

- 仅已支付成功且未退款订单可开票
- 支持电子发票
- 支持批量选择订单进行开票

### 7.5 成就规则

- 成就按账号累计，不因页面切换丢失
- 已解锁勋章默认永久保留
- 分享文案随当前成就动态变化

## 八、核心流程

### 8.1 新用户首次使用流程

1. 用户进入首页
2. 用户可先下载 My tokens 和预配置 AI 工具
3. 当用户发起订阅或充值时，系统引导其登录 / 注册
4. 用户进入订阅或充值页面完成首次购买
5. 直接选择平台内置模型发起首次调用，无需额外配置
6. 系统按订阅今日可使用次数或可使用余额扣费
7. 用户回到首页查看消耗变化
8. 如需在外部 AI 工具中使用其他模型，再到系统内补充供应商 API Key
9. 参考工具接入说明完成外部工具配置，由系统处理地址适配和协议转换

### 8.2 外部模型接入流程

1. 用户下载 My tokens 并下载已预配置好的 AI 工具
2. 用户不买平台套餐，也不充平台余额
3. 当用户准备新增供应商时，系统引导其登录 / 注册
4. 用户进入模型供应商页面新增供应商
5. 用户填写供应商名称、官网地址、请求地址、API Key
6. 系统验证信息有效性
7. 验证通过后自动拉取该 API Key 支持的全部模型
8. 用户在新增流程中最多选择 4 个模型完成首轮接入
9. 系统完成协议适配，使模型可在不同 AI 工具中被调用
10. 若其他工具需要更多模型，用户可在 AI 工具管理中继续补充配置

### 8.3 订阅续费流程

1. 用户在首页或个人中心看到订阅今日可使用次数不足
2. 点击进入订阅页
3. 选择套餐并支付
4. 若开启自动续费，系统记录续费授权状态
5. 支付成功后订阅资格生效
6. 同步更新订阅身份与对应勋章展示

### 8.4 余额充值流程

1. 用户在首页或调用失败弹窗中收到余额不足提示
2. 跳转到按量充值 Tab
3. 选择充值档位并支付
4. 余额到账
5. 用户可继续调用平台内置模型

### 8.5 开票流程

1. 用户进入订单记录
2. 筛选可开票订单
3. 选择单个或多个订单
4. 提交发票抬头、税号、邮箱等信息
5. 系统生成电子发票并发送到邮箱

## 十二、总结

My tokens 一期不是单纯做一个充值页，而是把账号、购买、调用和接入这几件事放到一个页面里讲清楚。

只要一期能把“买得到、接得上、用得明白、额度不足能回流”这条链路跑顺，这个系统就有明确价值。

用户最终只需要搞明白三件事：自己还能不能用，额度不够时去哪里补，怎么把平台能力接到常用工具里。
