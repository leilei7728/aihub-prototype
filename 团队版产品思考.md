# 智擎 Token 管理系统团队版产品思考

## 一页结论

### 1. 这个系统应该收费，而且不应该按“页面工具”收费

如果这个系统只是一个“帮员工统一登录几个大模型账号”的工具，价值不够强，不值得单独收费。

但如果它解决的是下面 4 件事，它就具备明确收费基础：

- 公司购买的模型账号、API Key、客户端配置不再绑定员工个人
- 员工入职、离职、转岗时，AI 使用权限可以快速分配和回收
- 老板和管理者能看到谁在用、用了多少、花了多少钱
- 多个 AI 工具和多个模型来源统一接入，避免重复采购和权限失控

这时用户买的不是“一个管理后台”，而是“公司级 AI 使用权管理能力”。

### 2. 真正愿意付费的，不是个人用户，而是小中型团队

优先目标客户不是 C 端个人，而是这类团队：

- 10 到 100 人，已经开始常态化使用 Claude、ChatGPT、Cursor、Gemini、API Key
- 没有能力或预算直接采购完整企业版
- 已经出现账号分散、成本不清、员工离职带走配置、多人重复买号等问题

这类团队对“统一管理、额度分配、离职回收、成本可视”有现实需求，也更容易接受按团队收费。

### 3. 收费逻辑应该从“卖 Token”升级成“卖管理能力 + 通道能力”

建议把产品拆成两层价值：

- 第一层：管理能力收费
适合已经自带模型预算和 API Key 的企业客户，核心卖点是账号、权限、审计、额度分配、人员流转管理。

- 第二层：通道能力收费
适合希望直接通过平台购买和使用模型能力的客户，平台既提供管理，也提供统一路由、协议适配和计费。

### 4. 最合理的早期收费模式

建议优先采用“基础 SaaS 订阅 + 调用抽成/充值毛利”双模式：

- 团队版基础费：按企业或按成员数收费
- 平台代充 / 平台路由调用：在模型调用或充值中获得毛利

这样做的好处是：

- 即使客户自带 Key，平台也能靠管理能力收费
- 即使客户不买管理版，只用平台路由，也能产生收入
- 后续可根据客户成熟度自然分层，不会把收入完全绑死在一种模式上

## 核心问题

老板现在看到的问题，表面上是：

> 公司买了一堆大模型账号，员工各用各的，离职带走，入职重开，系统需不需要收费？

它背后其实是一个更大的企业管理问题：

### 1. 资产归属不清

公司买的模型账号、API Key、桌面工具配置，经常绑在员工个人邮箱、个人电脑、个人支付方式上。

结果是：

- 公司付钱，员工持有
- 离职难回收
- 新人要重新配置
- 老账号闲置但没人敢删

### 2. 成本不可见

很多团队根本不知道：

- 哪些人在高频用 AI
- 哪些工具最烧钱
- 哪些账号闲置
- 哪些模型投入高但产出低

如果看不见成本，就无法管理预算，也很难向老板证明 ROI。

### 3. 权限分配粗糙

不是每个人都需要同样的模型能力。

现实里常见的两种极端都不好：

- 全员平权，谁都能用最贵模型，成本快速失控
- 账号共用，权限混乱，安全和审计都过不了

### 4. 人员流转断裂

员工入职、转岗、离职，本来应该像企业软件座席一样管理，但 AI 工具现在大多还是“个人账号逻辑”。

老板真正想买单的，是“人变了，公司的 AI 能力不丢、不乱、不重复买”。

---

## 为什么可以收费

### 1. 因为它解决的是管理问题，不是简单工具问题

只要产品能把“公司买的 AI 资源从个人手里收回到公司手里”，它就不是附属小工具，而是企业的基础管理能力。

这类能力天然可以收费，因为它直接带来：

- 成本节省
- 权限收拢
- 风险降低
- 交接效率提升

### 2. 因为它替代了企业内部大量隐性人工成本

很多团队今天是靠这些土办法在撑：

- Excel 记录账号
- 行政手动分配和回收
- 员工自己保存 Key
- IT 临时帮忙配环境
- 出问题再回头查谁在用

这些人工管理动作本身就有成本，而且很容易出错。

### 3. 因为它不是一次性需求，而是持续需求

只要团队持续使用 AI，这些管理动作就会持续发生：

- 新人入职分配
- 老人离职回收
- 额度调整
- 工具更换
- 模型切换
- 月度复盘

持续发生的问题，适合做持续订阅收费。

---

## 什么情况下不适合收费

下面这些情况，单独收费会比较弱：

### 1. 只提供账号聚合展示

如果系统只能展示余额、展示消耗、展示页面入口，本质上只是个 UI 层，收费说服力不强。

### 2. 不能真正收回资源控制权

如果员工仍然直接持有原始账号、原始 Key、原始客户端配置，公司后台只是“看一眼”，那它解决不了老板最痛的问题。

### 3. 没有成本管理闭环

如果看不到成员、部门、工具、模型的消耗拆解，也没有额度和权限控制，就很难被当成企业管理产品采购。

结论是：

不是“这个系统能不能收费”，而是“它有没有做到企业级管理深度，做到就能收费，做不到就只能当附赠功能”。

## 建议的产品定位

### 定位一句话

帮助中小团队统一管理 AI 账号、模型接入、调用额度和员工权限，让公司购买的 AI 能力可分配、可回收、可审计。

### 不是卖什么

- 不是卖一个充值页
- 不是卖一个账号收纳箱
- 不是卖单纯 API 转发

### 是在卖什么

- AI 资产管理
- 人员权限管理
- 使用成本管理
- 模型接入与协议适配

## 目标客户

### 第一优先客户

- 10 到 100 人团队
- 研发、设计、内容、电商运营、短剧、广告投放、出海团队
- 已经在混合使用网页端账号、桌面客户端、CLI 工具和 API Key

### 典型购买者

- 创始人 / 老板
- 行政采购负责人
- IT / 内部工具负责人
- 部门主管

### 典型使用者

- 开发
- 设计
- 运营
- 内容制作
- 投放人员

购买者和使用者不是同一类人，这意味着产品必须同时满足：

- 管理者看得懂 ROI
- 员工用起来不麻烦

## 建议收费模式

### 模式一：团队订阅费

适合“客户自带 API Key，只需要管理能力”的场景。

可选方案：

- 按团队收费：例如 199 元 / 月起，含 20 人
- 按成员阶梯收费：20 人内一个价，50 人内一个价，100 人内一个价
- 按功能版本收费：基础版、专业版、企业版

适合卖点：

- 成员管理
- 权限分配
- 离职回收
- 使用统计
- 审计记录

### 模式二：平台调用 / 充值收益

适合“客户直接通过平台买额度或走平台路由调用”的场景。

收入来源：

- 充值价差
- 路由服务费
- 协议转换与统一接入带来的毛利

适合卖点：

- 不想自己维护多个模型供应商
- 希望统一采购和统一支付
- 希望开箱即用

### 模式三：混合收费

推荐优先考虑混合模式：

- 基础管理订阅费，保证稳定收入
- 平台调用收益，增加弹性收入

这是更稳妥的商业模式，因为不把营收全部押在 Token 差价上。

## 建议版本划分

### 1. 免费版

目的不是赚钱，而是拉新和验证。

建议仅提供：

- 少量成员体验
- 基础模型接入
- 基础消耗查看
- 不开放高级权限和审计能力

### 2. 团队基础版

建议作为主售卖版本。

建议包含：

- 成员管理
- 工具接入管理
- 模型供应商统一管理
- 额度分配
- 基础消耗统计
- 离职回收

### 3. 团队专业版

建议包含：

- 部门维度统计
- 权限模板
- 审计日志
- 自动停用 / 自动提醒
- 更细粒度模型权限
- 批量导入导出

### 4. 企业版

适合更大客户，按项目谈。

建议包含：

- 私有部署或专属环境
- SSO / 企业身份体系接入
- 审批流
- 合同和专属服务

## MVP 建议

如果要尽快验证这件事能不能成，不建议一开始做大而全。

优先闭环建议如下：

### 1. 成员管理

- 添加成员
- 停用成员
- 离职回收
- 角色分组

### 2. AI 资产统一接入

- 统一管理平台内置模型
- 统一管理外部供应商 Key
- 员工侧不直接暴露原始 Key

### 3. 额度与权限

- 按成员分配额度
- 按角色限制模型能力
- 按工具限制接入范围

### 4. 消耗可视化

- 按成员看用量
- 按模型看成本
- 按工具看消耗

### 5. 工具接入

优先接最常见的 2 到 3 类工具：

- Claude Desktop
- Cursor / Codex CLI / OpenCode
- 平台内网页端

只要这 5 件事能跑通，就已经足以验证收费合理性。

## 风险点

### 1. 如果只做“充值平台”，天花板会低

因为平台价值会过度依赖模型价差，容易被替代，也容易陷入价格竞争。

### 2. 如果只做“管理后台”，进入门槛不够强

因为一部分团队会觉得 Excel、共享账号、企业微信表格也能凑合。

### 3. 最好的路径是“管理 + 接入 + 成本闭环”同时成立

只有三件事同时成立，老板才会觉得：

- 这不是锦上添花
- 这是一个能帮公司长期省钱、降风险、提效率的系统

## 建议给老板的汇报口径

可以直接用下面这段：

> 这个系统可以收费，但前提不是把它做成一个简单充值页，而是做成一个团队级 AI 账号和调用管理系统。公司真正愿意付费的原因，不是因为它能展示余额，而是因为它能把公司买的 AI 能力从员工个人手里收回来，实现统一分配、离职回收、成本可视和权限控制。早期建议采用“团队订阅费 + 平台调用收益”的双模式，先切 10 到 100 人、已经开始大量使用 AI 但还没有企业级管理方案的小中型团队。

## 对现有 My tokens 的启发

当前 My tokens 一期更偏个人版逻辑，重点是：

- 订阅
- 充值
- 模型接入
- 工具配置
- 个人消耗查看

如果要承接老板现在这个问题，后续需要自然长出团队版能力：

- 成员管理
- 团队额度
- 角色权限
- 成员维度统计
- 离职回收
- 团队资产统一归属

也就是说，后续可以形成两条产品线：

- 个人版：解决个人用户购买、接入和调用问题
- 团队版：解决公司管理、分配、回收和审计问题

这两条线可以共用底层模型接入和计费能力，但前台叙事和付费对象要分开。
